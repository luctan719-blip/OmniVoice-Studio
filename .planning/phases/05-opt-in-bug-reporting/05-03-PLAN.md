<objective>
Close Phase 5 by validating the reporter against real historical bugs, expanding the allow-list with anything the dry-run reveals as missing, locking down cross-platform `openUrl` parity in CI, and writing the Phase 2 handoff contract so subprocess-engine crashes carry full stack context when Phase 2 ships.

Purpose: 05-RESEARCH.md's "Compounding Effect" section makes the case that Phase 5 is the support-scaling lever for v0.3.x — but ONLY if the allow-list captures what maintainers actually need. The dry-run is the empirical check. We pick 3 representative historical issues, ask "would the Phase 5 payload have been enough?", and iterate the allow-list until the answer is yes for all three. Then we lock cross-platform parity in CI so a Tauri-2 plugin regression on macOS or Windows or Linux fails the PR. Then we hand the Phase 2 planner the contract for child-side error capture so when SubprocessBackend ships, subprocess crashes don't lose their stack trace.

Output: BUG-REPORT-HANDOFF.md + DRY-RUN.md + ci.yml extension + cross_platform test file + ALLOWED_FIELDS expansion (with tests). One human-verification checkpoint at the end.
</objective>

         ```json
           "ts": <unix seconds, float ok>,
           "pid": <int>
         }
         ```
      4. **Contract — installation pattern** — child must install `sys.excepthook` (NOT `signal.signal` — those are async-signal-safe constraints, different beast) at boot. Copy the exact snippet from 05-RESEARCH.md "Subprocess-Engine Error Capture" lines 785–805. Chain to `sys.__excepthook__` after writing so default behavior (printing to stderr) is preserved.

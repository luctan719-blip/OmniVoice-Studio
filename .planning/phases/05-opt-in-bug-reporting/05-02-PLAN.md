threat_model:
  - id: T-05-07
    category: Information Disclosure
    component: BugReportDialog preview <pre> block
    disposition: mitigate
    mitigation: "Dialog displays the Markdown source returned by /preview verbatim — same string the user is about to send to github.com. No Markdown rendering, no syntax highlighter, no parser that could re-interpret an injected pattern. The preview IS the payload (Pitfall #3 closure — backend redaction is the last line of defense; UI only displays what backend produced). Test asserts the rendered text equals the response body byte-for-byte."
  - id: T-05-08
    category: Tampering
    component: frontend/src/utils/bugReport.js opener.openUrl call
    disposition: mitigate
    mitigation: "openUrl receives ONLY the URL returned by /api/bug-report/preview (issue_url, search_url, local_log_path). The frontend never constructs the github.com URL itself — it cannot substitute a different host. Test asserts the URL passed to openUrl exactly matches the response from /preview (no client-side concatenation)."
  - id: T-05-09
    category: Spoofing
    component: PrivacyPanel toggle PUT
    disposition: accept
    mitigation: "PUT enforcement is backend-side (loopback Host check — T-05-05 in Plan 05-01). The frontend just calls fetch; if a malicious page running outside Tauri tried to flip the toggle, the backend rejects with 403. Frontend defense-in-depth not required here."
  - id: T-05-10
    category: Denial of Service
    component: Rust panic hook
    disposition: mitigate
    mitigation: "Pitfall #7 explicit handling: use take_hook() to capture the prior hook, install ours, and call the prior hook at the end. tauri-plugin-log's panic logging continues to work.

actionlint pinning policy (per 06-RESEARCH Package Legitimacy Audit + Phase 0 Pitfall 4):
- Pin to a commit SHA, NOT a floating tag.
- At execution time, run `gh release view --repo rhysd/actionlint --json tagName,publishedAt --jq '.tagName + " " + .publishedAt'` to get the latest stable release.
- Then run `gh api /repos/rhysd/actionlint/git/refs/tags/<tag> --jq '.object.sha'` to get the SHA.
- Use `uses: rhysd/actionlint@<full-sha>` in the YAML.
- Add a comment `# pinned to vX.Y.Z (release date)` so future maintainers can read what version corresponds to the SHA.

Parity check (option B) — single shell-script step (per 06-RESEARCH "Workflow-parity lint pattern"):
```
OFFENDERS=$(grep -nE 'tsc --noEmit' .github/workflows/ci.yml .github/workflows/release.yml \
  | grep -v 'bun run typecheck:ci' || true)
if [ -n "$OFFENDERS" ]; then
  echo "::error::Direct tsc --noEmit invocations bypass package.json's typecheck:ci script."
  echo "$OFFENDERS"
  exit 1
fi
```
The `bun run typecheck:ci` exclusion is the blessed single-source entry point established by PR #84.

Tag-shaped dry-run guard (option C) — per-job guard that lets PR-event runs short-circuit before publishing steps:
- Trigger extension: add `pull_request:` block to `on:` with the release-relevant paths above.
- Per-job guards (added at TOP of each downstream job that would publish, sign, or upload): `if: github.event_name != 'pull_request' || startsWith(github.head_ref, 'release/')` OR for the simpler pattern, every "publish" / "sign" / "softprops/action-gh-release" step gets `if: github.event_name == 'push' && startsWith(github.ref, 'refs/tags/v')` (this pattern is already used at `release.yml:498` for the checksum-publish step — extend the same pattern to every step that writes to the GH Release).
- Build/test jobs (matrix, smoke, typecheck, lint) run as-is on PR — that's the dry-run value.
- Secrets `TAURI_SIGNING_PRIVATE_KEY` and `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: GitHub already disables secrets on PR runs from forks; for same-repo PRs they're available — confirm publish steps gate on `event_name == 'push'` so secrets are never consumed on PR. Per 06-RESEARCH STRIDE row "Secret exfiltration via malicious PR": verify by reading release.yml end-to-end.
